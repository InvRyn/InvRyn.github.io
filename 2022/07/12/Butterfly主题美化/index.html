升级建议

为了减少升级主题后带来的不便，请使用以下方法（建议，可以不做）。

在 Hexo 的根目录创建一个文件 _config.butterfly.yml，并把 butterfly 主题目录的 _config.yml 内容复制到_config.butterfly.yml 去。( 注意: 复制的是主题的 _config.yml，而不是 hexo 的 _config.yml)

注意：不要把主题目录的 _config.yml 删掉
注意：只需要在 _config.butterfly.yml 进行配置就行；使用 _config.butterfly.yml，配置主题的 _config.yml 将不会有效果。

Hexo 会自动合并主题中的 _config.yml 和 _config.butterfly.yml 里的配置，如果存在同名配置，会使用 _config.butterfly.yml 的配置，其优先度较高。

使用 hexo douban 命令即可生成指定页面，如果不加参数，那么默认参数为-bgm。

需要注意，当安装了 hexo douban 之后，就不能用 hexo d 了，因为 hexo douban 跟 hexo deploy 的前缀都是 hexo d。在执行 hexo deploy 命令之前，需先执行 hexo douban -bm

如果配置了builtin参数为true，那么除了可以使用hexo douban命令之外，hexo g或hexo s也内嵌了生成页面的功能。

注意事项

之前我用此插件时，遇到 0 books have been loaded in 934 ms, because you are offline or your network is bad 的问题，最后 Node.js 版本的问题.

答案见评论区：https://github.com/mythsman/hexo-douban/issues/57

问题关键在于 Nodejs 版本的问题，之前的版本过高，按照评论区的说法，更换了 v12.20.2 的版本。更换完后再尝试爬取数据，即可了。

补充：更换 Node.js 版本的方法网上有很多简单的方法。我就很直接粗暴，把之前 v16.16.0 的版本直接卸载，又下载了一个 v12.22.12 的版本。

从零开始搭建 Hexo 博客

前言
写此篇文章的初衷就是记录自己用Hexo搭建个人博客的步骤，方便自己之后查阅学习

自媒体时代，每个人都有展示自己的舞台。
博客，我认为是记录并存储学习、心得、笔记的平台，不太需要动态博客的高可互动性。而国内越来越封闭的内容生态，越来越严格的文章审查，拥有自己的静态博客也是极好的事！

当前有许多开源的博客框架，比如Hexo、WordPress、Halo、Hugo、VuePress等，Wordpress由于是动态博客框架所以不能建在GitHub静态博客上，相比之下，我选择了Hexo。一是 Hexo 在国内流行，学习资料多，主题丰富；二是在尝试 Hugo未能成功在 GitHub 部署上后，断然选择Hexo（之前有成功部署经验)。

安装环境

安装下载非常简单，下一步即可。需要注意的是添加完成后需要添加环境变量，一般情况下会默认添加，如果没有添加，可手动添加。

安装 Node.js

官网链接：https://nodejs.org/en/

安装 Git

官网链接：https://git-scm.com/
下载合适的版本，一路 Next，到了 Select Components 界面，勾选 Git Bash Here 和 Git GUI Here。然后一路 next，完成安装。

检查是否正确安装
打开 cmd，输入 node -v 与 git –version，如果能输出信息则表示成功

【初始化一个 Hexo 博客】

  • 在任意位置创建一个文件夹，你的博客将出现在这个文件夹
  • 用命令行进入该文件夹，鼠标右键选择 Git Bash Here，进入此文件夹
  • 输入命令 hexo init + [ 刚才创建的文件夹名 ] ，程序会自动克隆一个 Hexo 项目到你当前目录，安装完成后依旧使用 hexo -v 查看版本号

Github 网站托管

创建 repository

打开 GitHub 官网，登录账号，点击 Create repository 新建一个储存库，输入储存库的名字，储存库的名字一定要和你自己账号的名字一样，后面一定要加**.github.io**，选择 Public 公开储存库，最后 Add a README file 添加 README 文件的勾也要勾上，点击 Create repository 创建储存库就完成了。

进入 repository 后点击 settings，往下滑找到GitHub pages，将 Source 里的 Branch 分支改成master，点击 Save 就好了。

Hexo 之禅

【忠告】
Hexo 主题的挑选和域名的挑选一样重要，请一定要慎重考虑！优先决定主题可以避免很多未来的反复折返与文章内容的修改适配主题。

《 Hexo 之禅 》：

  • 折腾主题是Hexo的最大原罪。专注于创作，而不是令人眼花缭乱的效果。名家之博，不假于主题之花哨，而得知于内容
  • 样式千万种，花哨莫去求。主题若彩虹，遇上方知有
  • 有问题记得首先去查看 Hexo 官方文档，善用搜索引擎
  • 善于分享，写作是检索你对知识理解程度的手段，而分享是你继续创作的动力
  • 不忘初心，方得始终

尾语
Hexo 目前有上百个主题，其中绝大部分都是免费开源并且长期有作者维护的，你可以在 Hexo-Theme 挑选喜欢的主题。本站选择 Butterfly 主题。
